化璟钰的站点之已知问题和计划

简介

这个站点的方向是静态网站。因为静态网站兼容性和操作性好（特别是 DOM）。ASP 似乎只兼容 Windows，PHP 不可信。不过如果我哪天疯掉了真的准备改为动态网站，那么我会使用 ASP。

还有一个方向，就是兼容性。一切页面都考虑所有情况，包括但不限于旧的浏览器和设备、关闭 JavaScript 等情况。所以部分功能会尽量在 JavaScript 不可用和浏览器不支持框架的情况下使用纯 HTML 方案，如果纯 HTML 无法实现，则暂时放弃功能。

这里的已知问题大多数是我在网站制作过程中遇到的一些问题，还有一些 Bug，而可能会比较少列出户体验问题。（因为我都是写到近乎完美再发布的。）如果你有我可能漏下的用户体验的问题或其它 Bug 修复甚至错别字，你可以发邮件或者打电话过来。相信你能找到。

但是类似不好看的问题，很抱歉我很可能不会妥协。（应该。）毕竟还要兼容旧的设备和浏览器，而且特效太多容易卡……

问题列表

  • CSS 和 HTML 的布局的代码有点乱
  • 无法使子页面的诸如标题、底栏、导航栏等元素和 CSS、JavaScript 的引用跟随指定模板变化，暂时只能在每个页面都写上对应的代码。
  • 暂时只能使用绝对定位实现导航栏的竖排模式。
  • 部分旧版本的 Internet Explorer 可能会出现背景完全透明的情况。
  • 暂时还没有能够正确实现部分页面在无工具栏的窗口中打开。
  • 文章列表没有分页，也无法自动分页。文章太多时加载列表速度慢。
  • 站点暂时不支持多语言，目前暂时未找到多语言适配和兼容性共存的解决方案。（不过当年做旧版倒是使用了多个子文件夹作为多语言的网页文件……）
  • 站点只支持 HTTP，不支持 HTTPS。容易被篡改。GitHub 貌似是 HTTPS，但是大概是由 GitHub 保管文件并由他们的证书认证。
  • 在外网没有购买域名。所以暂时用的是 github.io 结尾的域名。
  • 由于是 GitHub 的空间，不确定能不能搞动态网页，在有条件时会考虑购买国际版 Azure 云服务器顺便当云电脑玩。

计划列表

  • 适当调整代码和布局，方便后期修改时不会搞的很乱。
  • 可更换（也可禁用所有样式表）的主题和用户自定义样式表与主题。准备制作枪弹辩驳风格和 We Are* 风格的主题。当前的主题是秋之回忆风格和星之梦风格的主题。
  • 全局背景音乐，即使切换页面也不会中断播放。（但是我不想用框架和 JavaScript 啊！）
  • 将文章列表按时间整理。也有可能对文章检索单独制作一个使用 ASP 或 JavaScript 技术的页面。如果我疯掉了，可能会打开 document 的目录浏览功能。
  • 留言板，必须使用 ASP。考虑将留言板写在框架中并链接到另一服务器。（这两点倒可以妥协，毕竟留言板嘛。必须要保存数据的。看来得用 mklink 保存到 OneDrive 上了。）
  • 购买域名并换用独立 IP？前者估计很简单。后者……我自认为考虑 IPv4 独立 IP 是在做梦。
